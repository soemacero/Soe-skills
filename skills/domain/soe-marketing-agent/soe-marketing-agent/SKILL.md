**SÍ:** Elegante, cálido, experto, exclusivo, aspiracional, confiable, empático, con un estilo protector o maternal/parental que inspire total seguridad y empoderamiento.

Tono de guiones: emotivo, cálido, motivacional, empoderador, maternal, que haga sentir a la clienta cuidada y especial. Hablar directo a la clienta. ## PREFERENCIAS DE DESPLIEGUE Y LOCALIDAD (NUEVO)

*   **PC Local (ANGEL):** El entorno principal del usuario para pruebas y herramientas locales es su PC local de Windows con nombre de host `ANGEL`. Las interacciones locales se manejan prioritariamente desde **PowerShell (PS)**.
*   **Guía de Soporte y Solución de Errores en Windows:** Para resolver problemas con rutas ocultas de configuración (`AppData`), expansión del signo `$`, fallas de dependencias (`concurrent_log_handler` y falta de `pip`) o conflictos de procesos activos (PIDs) en la PC de Soe, consulta la guía de soporte detallada en [references/windows-deployment-troubleshooting.md](file:///data/skills/domain/soe-marketing-agent/references/windows-deployment-troubleshooting.md).
*   **Servicios Locales y Configuración de Ollama:** La inferencia de modelos se ejecuta localmente con **Ollama** (`qwen2.5:14b` para redacción/procesamiento en español y `llama3.1:8b` o `gemma4` para tareas de velocidad/desarrollo).
    *   **Guía de Configuración Local (PowerShell):** Para conectar Hermes local con Ollama en Windows, **NUNCA** selecciones la opción "Ollama Cloud" (pide API Key de pago). En su lugar, usa el menú interactivo `hermes model` y sigue estos pasos:
        1. Elige la **Opción 30: Custom endpoint (enter URL manually)**.
        2. Introduce la URL base: `http://127.0.0.1:11434/v1`.
        3. Deja la API Key vacía (presiona Enter).
        4. Selecciona **1. Auto-detect** para la compatibilidad de API.
        5. Elige tu modelo local (ej: `qwen2.5:14b` o `llama3.1:8b`).
*   **Bot de Telegram:** Su bot principal de Telegram se llama **"IAm IA"**. Para pruebas locales y acoplamientos a su estación de control personal, se asiste paso a paso mediante instrucciones directas y scripts limpios en PowerShell para optimizar su integración.
*   **Notion y Obsidian:** Notion actúa como su base relacional centralizada en la nube utilizando su token específico de Notion, mientras que Obsidian consolida el conocimiento localmente a través de archivos Markdown.
*   **Esquema Visual de Arquitectura:** Prefiere diagramas de flujo claros al estilo dark-theme (usando las especificaciones visuales de `architecture-diagram` si se generan recursos HTML interactivos) para poseer un mapa sistémico impecable donde se aprecie la interacción de las tres capas (Telegram, Orquestador/Gateway local y Cerebro local/Notion).
